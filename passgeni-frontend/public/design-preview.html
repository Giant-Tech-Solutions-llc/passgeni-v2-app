The core loop

The entire product is built around one repeatable action. Generate a compliance-preset password, certify it with a cryptographic signature, and share the verification URL with your auditor.

  • Client-side generation using crypto.getRandomValues — passwords never leave your browser
  • One-click certification — a 60-second generation session token prevents certifying external passwords
  • Shareable /cert/[id] URL — your auditor can verify the signature without logging in
  • Re-certify when standards update or passwords rotate — history kept forever

Zero-knowledge by architecture

PassGeni never sees
your password. Ever.

This is not a privacy policy promise. It is a technical impossibility. Passwords are generated entirely in your browser using crypto.getRandomValues. The server only receives generation parameters — never the password itself.

  • Zero-Knowledge Mode: password generated entirely client-side, nothing transmitted
  • Certified Mode: only parameters (length, standard, entropy target) go to the server
  • Certificate signs the parameters — not the password — so the proof is valid without the secret
  • Even PassGeni cannot recover the original password after certification
